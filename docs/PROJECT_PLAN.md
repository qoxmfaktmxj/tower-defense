# 프로젝트 계획서

## 목표

이 프로젝트의 목표는 한국어 이용자를 대상으로 한 브라우저 타워 디펜스 게임을 만드는 것입니다.

핵심 목표:

- 짧은 시간 안에 플레이 가능한 MVP를 만든다.
- 웹 셸과 전투 런타임을 분리해 이후 확장이 가능하도록 한다.
- 한국어 UI와 빠른 반복 개발에 맞는 데이터 중심 구조를 만든다.

## 초기 원칙

- 전투는 Phaser가 책임진다.
- 웹은 라우팅과 패널 UI를 책임진다.
- DB는 API를 통해서만 접근한다.
- 타워, 웨이브, 맵은 데이터 파일로 관리한다.
- 한국어 문구를 기본으로 유지한다.

## 1차 계획

1. 모노레포 기본 구조 확립
2. 웹 셸, 게임 런타임, API 분리
3. 최소 플레이 가능한 전투 루프 완성
4. 결과 제출과 랭킹 흐름 연결
5. 맵/웨이브/업그레이드 확장
6. 실제 DB 연결 기반 마련

## 실제 반영된 방향

현재는 아래 방향으로 정리되어 있습니다.

- 전장 테마는 `프론트라인` 단일 테마 유지
- 맵 3종
- 웨이브 20개
- 타워 3종
- 업그레이드 5단계
- 레벨 5 특수 효과
- 공지/상점 제거
- 랭킹 중심의 단순한 웹 흐름 유지

## 왜 단일 테마로 정리했는가

모드 B/C처럼 대체 테마를 동시에 운영하면 다음 문제가 생기기 쉬웠습니다.

- 테스트 포인트가 급격히 늘어난다.
- 에셋과 밸런스가 따로 놀기 쉽다.
- UI와 실제 전장 상태가 엇갈릴 가능성이 커진다.
- 유지보수 비용에 비해 사용자 체감 이득이 작다.

그래서 현재는 `하나의 기본 테마를 안정화`하는 쪽이 더 맞다고 판단했습니다.
