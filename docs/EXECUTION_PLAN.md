# 실행 계획서

## 목적

현재 구조를 유지한 채 `프론트라인` 단일 테마의 완성도를 높인다.  
이번 실행 단계는 전장 연출, 씬 프레젠테이션, 오디오 파이프라인, 스코프 정합성 정리에 집중한다.

## 1차 실행 범위

1. 맵별 시각 정체성 강화
- `han-river-front`, `metro-grid`, `red-canyon` 각각에 배경색, 슬롯 색, 경로 가이드, 분위기 레이어를 부여한다.
- 전장별 전술 브리핑 데이터를 `StageDefinition`에 포함한다.

2. 씬 프레젠테이션 업그레이드
- `PreloadScene`에 전장 프리뷰, 위협도, 추천 포대, 작전 요약을 표시한다.
- `MenuScene`을 작전 브리핑 카드 중심으로 재구성한다.
- `ResultScene`에 전장 프리뷰와 전술 메모, 핵심 결과 요약을 표시한다.

3. 오디오 파이프라인 연결
- React에서 전달한 `volume` 값을 Phaser 런타임에서 실제로 반영한다.
- 절차형 오디오를 사용해 발사, 착탄, 웨이브 시작, 적 도달, 승리/패배 피드백을 추가한다.
- 전투 중 저강도 배경 펄스를 반복 재생한다.

4. 스코프 정합성 정리
- 현재 제품 범위에서 제외한 공지/상점/결제 관련 프런트 흔적을 정리한다.
- 사용하지 않는 목업 데이터와 쿼리 키를 제거한다.

## 후속 단계

1. `GamePage` 전용 HUD 레이아웃 재배치
2. 맵별 웨이브 개성 분리
3. 랭킹 신뢰도 로그와 서버 검증 강화
4. 자동 테스트 추가
   - 경제 계산
   - 웨이브 진행
   - `/game` 진입 및 설치/업그레이드 스모크

## 검증 기준

- `corepack pnpm typecheck`
- `corepack pnpm build`
- 브라우저 스모크
  - 로그인
  - 로비 진입
  - `/game` 진입
  - 작전 개시
  - 슬롯 선택
  - 타워 설치
